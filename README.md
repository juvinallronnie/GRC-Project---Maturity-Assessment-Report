CloudPulse Dynamics – Mock Access Management Maturity Assessment

Purpose of the Project:
This project evaluated the maturity of identity and access management (IAM) controls at CloudPulse Dynamics, focusing on onboarding, offboarding, privileged access, multi-factor authentication (MFA), and periodic access reviews. Using NIST SP 800-53 and the CMMI maturity model, the assessment determined current control effectiveness and defined target states to improve security, enforce least privilege, and reduce unauthorized access risks.

Steps Followed:
The assessment followed a structured approach:
1.	Defined scope across user provisioning, deprovisioning, privileged access, authentication, and access reviews. 
2.	Collected evidence of current practices, including manual onboarding, delayed offboarding, optional MFA, and unreviewed access. 
3.	Assessed each control’s maturity using CMMI levels, identifying gaps in onboarding (Level 1), offboarding (Level 2), privileged access (Level 1), and authentication (Level 2). 
4.	Established target maturity levels, such as onboarding at Level 3 and privileged access management at Level 4. 
5.	Developed recommendations for short- and long-term improvements, including standardized workflows, automation, RBAC, PAM tools, and periodic access reviews.
   
Results and Recommendations:
The assessment revealed critical weaknesses in access control, including manual processes, delayed deprovisioning, excessive permanent privileges, inconsistent MFA, and no formal access reviews.

Key recommendations included:
 •	Short-term: Standardize onboarding and provisioning, enforce least-privilege access, and formalize account requests. 
 •	Long-term: Automate onboarding/offboarding integrated with HR systems, implement RBAC, deploy privileged access management tools, establish quarterly access reviews with audit logs, and track metrics for continuous improvement. 
These improvements enhance security posture, operational efficiency, and compliance alignment.

What I Learned:
This project showed me that effective IAM needs both clear processes and strong controls. I gained hands-on experience using NIST SP 800-53 and CMMI to assess maturity, identify risks, and prioritize improvements. It also reinforced the importance of aligning security controls with business goals and implementing practical, measurable solutions to reduce access risks across the organization.
